# Dataria — Contexto de producto (para humanos y para IA)

Este documento existe para que cualquiera (o cualquier sesión de Claude Code) que trabaje en este repo entienda quién es el cliente y qué estamos vendiendo, sin tener que reconstruir esto de cero cada vez. Se actualiza a medida que el negocio se define mejor — es un borrador vivo, no una verdad fija.

## Qué es Dataria
Herramientas con IA para pymes y autónomos. Automatizan gestión y decisiones operativas sin requerir un equipo técnico interno.

Tagline: **"Menos datos sueltos. Mejores decisiones."**

## Rubros / verticales (5 demos en el sitio)
- Gastronomía — calculadora de food cost
- Distribuidoras — rutas + mapa
- Talleres — kanban de órdenes
- Inmobiliarias — CRM pipeline
- E-commerce — suite de 5 módulos

**Estado (jul-2026): ningún rubro tiene todavía tracción de cliente validada por encima de los demás.** Los 5 están en etapa de validación pareja. No asumir que gastronomía es "el" foco solo porque hay más logos de ese rubro en el sitio — eso todavía no está confirmado como la prioridad de negocio.

## Cliente objetivo
Pymes y autónomos — **rango amplio a propósito**. No es solo "el dueño de un local chico sin equipo técnico" (eso excluye pymes con algo más de estructura), pero tampoco apunta a empresas grandes con área de sistemas propia. El común denominador es: el que decide no tiene equipo técnico dedicado y necesita algo simple de adoptar.

## Contra qué compite (mix, no una sola cosa)
- Procesos manuales / planillas de Excel (el cliente hoy no usa ninguna herramienta)
- Herramientas específicas ya existentes en cada rubro (el cliente ya usa algo y hay que desplazarlo)

No asumir un solo tipo de competencia al escribir contenido, copy de Ads, o preguntas de auditoría GEO — contemplar ambos escenarios.

## Call to action actual
Consulta gratuita de 15 minutos, o ver demos en vivo. Nunca se muestran precios en el sitio — todo redirige a CTA de conversación.

## Cómo usar este documento
- Para auditorías GEO (qué le preguntamos a Gemini/ChatGPT para ver si citan a Dataria): usar personas dentro del rango "dueño o gerente de pyme sin equipo técnico", cruzando los 5 rubros, y frasear tanto desde "no uso nada, todo a mano" como "uso [herramienta X del rubro], ¿hay algo mejor?".
- Para contenido y copy: evitar lenguaje que excluya pymes con algo de estructura ("solo para changuitos") ni lenguaje corporativo que espante al dueño de un local chico.
- Para Ads: el objetivo de negocio actual no es "vender un plan" sino generar consultas — el CTA principal siempre es la reunión de 15 minutos.
